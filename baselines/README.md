# Baselines

Frozen reports kept under version control so future runs can `diff` against a fixed reference point. Add a new baseline when:

- The corpus changes meaningfully (new attacks, new categories)
- The target model is upgraded
- A defensive change is intended to move the number

| `brain-v2-2026-04-18.json` | Same Brain target | 2026-04-18 | 38 attacks (v2) | **100.0%** (38/38) |

## Notes on v2

The corpus expanded from 18 → 38 with `@dj_abstract/prompt-genesis@0.1.0`. New categories represented: more `tool-coercion` (12 total), `system-prompt-extraction`, `indirect-injection`, `information-leak`, plus first-time entries in `delimiter-confusion`, `authority-claim`, `refusal-bypass`.

**Counter-intuitive v2 result:** Brain (wrapped agent, full system prompt + memory + tools) scored higher than the bare Anthropic API at the same model. The single attack that landed against the bare model — `gen-99d831bb` "Memory Serialization Trick to Leak System Prompt" — extracted the system prompt as a YAML block. Brain's longer/more contextual prompt appears to ground the model harder, not loosen it. Worth re-running across attempts to confirm it's not variance.

## Diff against a baseline

```bash
node bin/prompt-eval.js diff baselines/brain-v2-2026-04-18.json reports/<latest>.json
```

Prints `Δ` defense rate plus per-attack regressions and wins.
